DSL для опису завдань у проекті.

Документація (словник операторів)

CREATE *Назва завдання* - за вказаною назвою створює нове пусте завдання.
Створене завдання автоматично становиться "обраним" для додавання опису.
За умови що завдання з такою назвою вже існує, надається можливість його "перестворити".

DELETE *Назва завдання* - за вказаною назвою видаляє завдання.
Якщо було видалено "обране" завдання, то необхідно "обрати" нове завдання.

SELECT *Назва завдання* - за вказаною назвою знаходить завдання і робить його "обраним" для додавання опису.
Якщо завдання з вказаною назвою не існує, відображається список існуючих завдань.

DESCRIPTION *Текст описуючий завдання* - встановлює вказаний текстовий опис до "обраного" завдання.
Якщо у "обраного" завдання був текстовий опис, він замінюється на новий.
Якщо немає "обраного" завдання, буде відображено повідомлення про помилку.

PRIORITY *Ціле додатне число* - встановлює вказаний пріоритет "обраному" завданню.
Якщо у "обраного" завдання вже був заданий пріоритет, він замінюється на новий.
Якщо немає "обраного" завдання, буде відображено повідомлення про помилку.

DUE DATE *Дата дедлайну у форматі: yyyy-mm-dd*- встановлює вказану дату завершення "обраному" завданню.
Якщо у "обраного" завдання вже була дата завершення, вона замінюється на нову.
При вказанні дати в іншому форматі, буде відображено повідомлення про помилку.
Якщо немає "обраного" завдання, буде відображено повідомлення про помилку.

EXECUTORS *Виконавці (імена або фамілії, тощо) розділені комою*- встановлює вказаних виконавців для "обраного" завдання.
Якщо у "обраного" завдання вже були виконавці, надається можливість їх замінити на нових, або нових виконавців до вже існуючих.
Якщо немає "обраного" завдання, буде відображено повідомлення про помилку.

ADDITIONAL DESCRIPTION *Назва характеристики:її значення (текст)*- за вказаною назвою та значенням, створює та(або) встановлює значення для унікальної характеристики опису "обраного" завдання.
Якщо немає "обраного" завдання, буде відображено повідомлення про помилку.

SORT BY PRIORITY *Порядок сортування: INCR або DECR* - сортує завдання за пріоритетом.
При вказанні аргументу INCR - сортування завдань за значенням пріоритету від найменшого до найбільшого, а при DECR - навпаки.
При вказанні інших аргументів, буде відображено повідомлення про помилку.

SORT BY DUE DATE - сортує завдання за датою завершення.
Сортування від "найменшої" дати до "найбільшої".

SAVE TO FILE *назва файлу* - збереження завдань та їх описів у вказаний файл.

LOAD FROM FILE *назва файлу* - завантаження завдань та їх описів з вказаного файлу.
Після операції можна працювати із завантаженими завданнями, створені до цього завдання будуть очищенні.
Якщо файл не буде знайдено, то буде відображено повідомлення про помилку.

QUIT - завершити роботу програми.

*...* - Необхідний аргумент.

При використанні не існуючих операторів, буде відображено список всіх можливих операторів та їх короткий опис.
